# ユースケース記述（ホテル予約システム）

最新版。実装（`hotel-reservation-system`）の各ユースケースに対応。

---

## UC2. 部屋を予約する

| 項目 | 内容 |
|------|------|
| 主アクター | 利用者 |
| 目的 | 利用者が希望の部屋タイプ・宿泊予定日で部屋を予約し、予約番号を得る |
| 事前条件 | システムが稼働している |
| 事後条件 | 予約が「予約中」で作成され、予約番号が発行される。対象の部屋が1室確保される |
| 基本フロー | 1. 利用者が部屋タイプと宿泊予定日を入力する<br>2. システムが該当タイプの空室を1室確保する（空室数を1減らす）<br>3. システムが予約を作成し、予約番号を発行する<br>4. システムが予約番号を表示する |
| 代替フロー | 2a. 空室が無い場合：システムは「空室がありません」と警告を表示し、処理を終了する |

## UC3. チェックインする

| 項目 | 内容 |
|------|------|
| 主アクター | ホテル受付係（利用者が予約番号を提示） |
| 事前条件 | 対象の予約が「予約中」で存在する |
| 事後条件 | 予約が「宿泊中」になり、宿泊記録が生成され、部屋が使用中になる |
| 基本フロー | 1. 受付係が予約番号を入力する<br>2. システムが予約を検索し、予約情報を表示する<br>3. 受付係が受付係名を入力して確定する<br>4. システムが予約を「宿泊中」にし、部屋を使用中にし、宿泊記録を生成する |
| 代替フロー | 2a. 予約が存在しない：エラーを表示して終了<br>4a. 既に「宿泊中」：「既にチェックイン済み」エラーを表示して終了 |

## UC4. チェックアウトする

| 項目 | 内容 |
|------|------|
| 主アクター | ホテル受付係（利用者が部屋番号を提示） |
| 事前条件 | 対象の部屋に「宿泊中」の宿泊記録が存在する |
| 事後条件 | アウト日時が記録され、支払い記録が生成され、部屋が空室に戻る |
| 基本フロー | 1. 受付係が部屋番号を入力する<br>2. システムが宿泊データを検索し、宿泊情報を表示する<br>3. 受付係が支払いを確定する<br>4. システムがアウト日時を記録し、基本宿泊料から支払い記録を生成し、部屋を空室に戻す<br>5. システムが支払い料金を表示する |
| 代替フロー | 2a. 宿泊データが無い：「宿泊データが見つかりません」エラーを表示して終了 |

## UC6. 予約をキャンセルする ★演習6（保守で追加）

| 項目 | 内容 |
|------|------|
| 主アクター | 利用者 |
| 事前条件 | 対象の予約が「予約中」で存在する |
| 事後条件 | 予約が「キャンセル」になり、確保していた部屋が空室に戻る |
| 基本フロー | 1. 利用者が予約番号を入力する<br>2. システムが予約を検索し、予約情報を表示する<br>3. 利用者がキャンセルを確定する<br>4. システムが予約を「キャンセル」にし、部屋を空室に戻す |
| 代替フロー | 2a. 予約が存在しない：エラーを表示して終了<br>3a. 既に「宿泊中」（チェックイン済み）：「チェックイン済みの予約はキャンセルできません」エラー<br>3b. 既に「キャンセル」：「既にキャンセル済み」エラー |
